5. 脉象组合 vs 张氏特色脉 (复合脉)

171 案含 zx_special (张氏特色脉), 这 171 案的 脉象组合往往是"特色脉 + 基础脉" 形式:

张氏特色脉常见基础脉组合案数

按之即无 (22)按之即无 + 沉 + 细, 按之即无 + 微22
弦长有力 (19)弦长有力 + 数, 弦长有力 + 弦19

弦硬而长 (10)弦硬而长 + 数, 弦硬而长 + 弦10
关键洞察: 张氏特色脉 = 复合脉的细拆。 例: "弦硬而长" = 弦 + 硬 + 长 三个基础脉的复合; "上盛下虚" = 寸盛 + 尺虚 两个分部脉的复合。 这印证了 v2.7.7.1 报告: 特色脉是"脉简"的工具, 不是反"脉简"。

阶段 1: 脉象组合提取 (已完成)

  • 输入: 994 案 (cases_v2.7.1.json)
  • 输出: 319 个唯一脉象组合 + 597 个含脉案 (本案完成)
  • 方法: pulse_chars 字段排序去重
  • 数据: /data/user/work/pulse_combos_v27.json

阶段 2: 脉-案索引库 (建库)

  • 输入: 319 组合 + 597 案 + 994 案全字段
  • 输出: 倒排索引, key = 脉象组合, value = 案例 ID 列表
  • 字段: 案例 ID, 父标题, 病名, 方剂, 5 维位置, 张氏特色脉, 脉案原文
  • 格式: combo_index.json

阶段 3: 脉-证-方三联标注

  • 输入: 597 案 + 78 张氏方剂库
  • 输出: 每个组合标注: 证型 (大气下陷 / 冲气上冲 / ...), 方剂 (升陷汤 / 镇肝熄风汤 / ...)
  • 方法: 提取 diagnosis 字段, 对应到 78 方剂
  • 目的: 让脉组合能直接给出"方剂"

阶段 4: 输入匹配 (前端)

  • 输入: 用户勾选脉象 (如 弦+数+细)
  • 匹配: 倒排索引查 597 案, 找出含此组合的案例
  • 排序: 按"完全匹配 > 子集匹配"排序, 按案例频次加权
  • 输出: Top 10 匹配案例 + 证型 + 方剂

阶段 5: 辨证输出 (产品)

  • 输入: Top 10 案例
  • 输出: ① 证型诊断 ② 张氏方剂 ③ 案例原文 ④ 加减建议
  • 形态: 已有的 step4match.html 升级, 加入 v2.7 组合库
  • 二期: 加入 5 维位置, 加入舌诊 / 问诊字段联合匹配
