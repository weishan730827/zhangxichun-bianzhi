⑥ 倒排索引原理

  1. 完整组合索引: 用户勾选 N 脉 → 查 combo_index["脉1|脉2|..."] → 直接得案例 ID 列表
  2. 子集匹配: 任何 N 脉的子集都查 signature_index (1-2 脉索引), 累加得分
  3. 脉+特色组合: 查 combo_index["+|脉|特色"]
  4. 方剂反向: 根据匹配案例统计方剂频次, 输出 Top 5 方剂
  5. 评分: 完全匹配 100 分, 子集匹配按 交集/并集 比
